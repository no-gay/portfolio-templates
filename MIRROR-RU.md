# Зеркало liaqil.space в зоне .RU — инструкция

Цель: поднять второе зеркало сайта в зоне `.ru`, чтобы:
- сайт стабильно открывался из РФ (GitHub Pages периодически «тормозит»/недоступен из России);
- появилось русское доменное имя для доверия клиентов (Avito, Kwork, визитки);
- оба домена обновлялись автоматически из одного git-репозитория.

> Текущая площадка: `https://liaqil.space` → GitHub Pages (CNAME уже прописан).
> Зеркало: `https://liaqil.ru` (пример) → российский хостинг, автодеплой из этого же репо.

---

## 0. Что нужно от пользователя (чек-лист)

Прежде чем настраивать, подготовьте:

- [ ] **Название домена в зоне .ru** — например `liaqil.ru`. Проверьте, свободен ли он (см. шаг 1).
- [ ] **Регистрация/вход у российского регистратора** — паспорт + телефон (требуется для `.ru` по договору).
- [ ] **Аккаунт на хостинге** (Beget / Timeweb Cloud — см. шаг 2).
- [ ] **Доступ к этому GitHub-репозиторию** — чтобы добавить секреты для автодеплоя (шаг 4).

> ⚠️ Домен `.ru` оформляется **только на физлицо/юрлицо с подтверждением** — нужен паспорт. Регистратор сам передаст данные в Координационный центр.

## 1. Проверка и регистрация домена .ru

### Свободен ли домен?
whois-проверка — бесплатно на сайтах регистраторов:
- https://www.nic.ru/whois/ (Ru-Center)
- https://www.reg.ru/whois/
- https://beget.com/whois

### Где купить (аккредитованные регистраторы РФ)

| Регистратор | Цена .ru (первый год) | Продление | Плюсы |
|---|---|---|---|
| **Beget** | ~199–290 ₽ | ~249 ₽ | Дешёвый, можно сразу хостинг там же |
| **Ru-Center (NIC.ru)** | ~270 ₽ | ~320 ₽ | Самый крупный, надёжный |
| **Reg.ru** | ~199 ₽ | ~299 ₽ | Удобный кабинет |

**Рекомендация:** домен + хостинг у одного провайдера (Beget или Timeweb) — меньше возни с DNS, иногда домен в подарок при оплате хостинга на год.

## 2. Выбор хостинга (рекомендация: Timeweb Cloud или Beget)

Сайт **статический** (HTML/CSS/JS без бэкенда), поэтому дорогой хостинг не нужен.

### ✅ Рекомендованный вариант — **Beget** (виртуальный хостинг)
- ~169–290 ₽/мес;
- **бесплатный SSL** Let's Encrypt (без доплат, автопродление);
- есть FTP/SFTP/SSH для автодеплоя;
- сайт-зеркало ложится прямо в `public_html`;
- работает из РФ без блокировок.

### Альтернатива — **Timeweb Cloud** (App/VPS)
- от 180 ₽/мес (виртуальный хостинг) или VPS от ~190 ₽;
- **бесплатный Let's Encrypt** через certbot (на VPS/Cloud);
- нативный **git-deploy** из репозитория;
- ⚠️ На *виртуальном* хостинге Timeweb Let's Encrypt = 99 ₽/мес (плата за управление). Поэтому для бесплатного SSL берите **Timeweb Cloud (VPS)**, а не обычный хостинг.

> ❌ **GitHub Pages для .ru-зеркала не подходит**: нестабильно из РФ, ради чего зеркало и поднимается.

### Итог выбора
| Ваш случай | Берём |
|---|---|
| Хочу «под ключ» и дёшево | **Beget**, тариф «Blog» или «Static» |
| Хочу бесплатный SSL и git-deploy «по-программистски» | **Timeweb Cloud VPS** + nginx + certbot |

## 3. Привязка домена к хостингу

После покупки домена:

1. В панели хостинга **создать сайт** → указать домен `liaqil.ru`.
2. В панели **регистратора домена** прописать **NS-серверы хостинга**
   (для Beget: `ns1.beget.com`, `ns2.beget.com` … — выдадут в письме/кабинете).
3. Дождаться делегирования — **от 15 минут до 24 часов**. Проверить:
   ```bash
   dig liaqil.ru +short
   # или https://check-host.net/check-ping?host=liaqil.ru
   ```
4. В панели хостинга **включить бесплатный SSL** (Let's Encrypt) для `liaqil.ru`. Сертификат выпустится автоматически после того, как домен начнёт резолвиться.

## 4. Автодеплой из этого git-репозитория

Цель: `git push origin main` → сайт обновляется и на `liaqil.space` (GitHub Pages), и на `liaqil.ru` (хостинг) — автоматически.

### 4.1. Что деплоить
В корне репо лежит **весь статический сайт** (`index.html`, папки шаблонов, `images/`, `styles.css`, `script.js`, `telegram-form.js`). Деплоим содержимое корня репозитория в папку сайта на хостинге (обычно `public_html/` или `liaqil.ru/public_html/`).

### 4.2. Вариант А — GitHub Action + SFTP (работает с любым хостингом)

Создать в репозитории файл `.github/workflows/deploy-ru.yml`:

```yaml
name: Deploy to .ru mirror

on:
  push:
    branches: [ main ]

jobs:
  deploy-sftp:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Upload to hosting via SFTP
        uses: wlixcc/SFTP-Deploy-Action@v1.2.4
        with:
          server: ${{ secrets.RU_SFTP_HOST }}      # напр. sftp.beget.com
          port: 22
          username: ${{ secrets.RU_SFTP_USER }}
          password: ${{ secrets.RU_SFTP_PASS }}
          local_dir: './'                            # весь сайт
          remote_dir: ${{ secrets.RU_SFTP_PATH }}   # напр. /liaqil.ru/public_html/
          delete: 'false'                            # true = удалять старые файлы перед заливкой
          exclude: '.git*,.github,*.md,take-screenshots.js,node_modules,promo-texts.md,MIRROR-RU.md'
```

### 4.3. Добавить секреты в GitHub
`Репозиторий` → **Settings** → **Secrets and variables** → **Actions** → **New repository secret**:

| Имя секрета | Значение | Пример |
|---|---|---|
| `RU_SFTP_HOST` | SFTP-сервер хостинга | `sftp.beget.com` |
| `RU_SFTP_USER` | логин от хостинга | `glebklimenko` |
| `RU_SFTP_PASS` | пароль от хостинга | `••••••••` |
| `RU_SFTP_PATH` | путь к папке сайта | `/liaqil.ru/public_html/` |

После следующего `git push` — GitHub Action зальёт сайт на `.ru`.

### 4.4. Вариант Б — git-deploy на Timeweb Cloud (без GitHub Actions)
Если выбрали Timeweb Cloud: в панели App-платформы подключить репозиторий → выбрать ветку `main` → указать root-папку `/`. Деплой автоматический при пуше. SSL подключается галочкой.

## 5. Зеркало = дубль. Что НЕ меняем

Зеркало — это **копия** основного сайта. Поэтому:

- **Никаких правок HTML под `.ru`** — контент, картинки, формы те же.
- **Telegram-форма** (`telegram-form.js`) работает с любого домена — токен бота один.
- **canonical / sitemap** — оставляем на основном домене `liaqil.space`, чтобы SEO не дублировался (см. шаг 6).

---

## 6. SEO-настройка зеркала (чтобы не было дублей)

Поисковики сочтут `liaqil.space` и `liaqil.ru` дубликатами. Чтобы основное доменное имя было главным:

В `index.html` и всех страницах шаблонов уже есть `<link rel="canonical">` —
убедиться, что он указывает на **основной** домен:
```html
<link rel="canonical" href="https://liaqil.space/" />
```
(на зеркале `.ru` canonical всё равно ведёт на `.space` — это правильно для SEO).

В `Яндекс.Вебмастер`:
1. Добавить **оба** сайта (`liaqil.space` и `liaqil.ru`).
2. У `liaqil.ru` указать **главное зеркало** = `liaqil.space`
   (раздел «Индексирование» → «Главное зеркало»).
3. Подтвердить права (мета-тег или HTML-файл).
